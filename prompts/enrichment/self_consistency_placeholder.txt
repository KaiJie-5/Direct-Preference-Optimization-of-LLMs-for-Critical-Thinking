Rules for reflective_question_candidates:
Return 3 to 5 reflective questions.
Each question must be connected to either:
- at least one code_id in candidate_code_matches, or
- at least one provisional_code_id in possible_new_codes.

Rules for code_quality_examples:
Return exactly these four keys:
- wrong_code: A code that does not match the target segment data.
- descriptive_not_answering_research_question: A code that is true of the segment but not useful for the study aim.
- too_broad_code: A code that relates to the segment but is too general.
- useful_analytical_code: A code that is grounded in the segment and helps answer the research question(s).
These examples are teaching contrasts, not final applied codes.

Rules for evidence quotes:

Do not quote from previous_context or next_context as the main evidence unless the target segment is unclear.

Copy the Target segment text exactly into analysis_unit.target_text.
